# ペルソナ定義

## ペルソナ 1: 一般ユーザー

| 項目 | 内容 |
|------|------|
| **名前** | 田中 健太（仮名） |
| **年齢** | 32歳 |
| **職業** | IT企業 プロジェクトマネージャー |
| **ITリテラシー** | 高い（日常的にWebサービス・アプリを利用） |

### 動機
- 新しいスキルや知識に興味はあるが、何を学べばいいか明確ではない
- 検索して探すのは面倒。偶然の出会いで「これ面白そう」と思いたい
- 通勤中や休憩時間にスマホで気軽に眺めたい
- 自分が知らなかった分野に触れて視野を広げたい

### 課題
- 研修コースが多すぎて選べない
- 検索しても自分が知っているキーワードでしか探せない
- まとまった時間がないと学習に取り組めないと思い込んでいる
- 興味を持ったコースを後で見返せない

### ゴール
- 隙間時間に没入的にコースを眺めて、新しい発見をする
- 気になったコースをコレクションして、学びの幅を可視化する
- 「気がつけば30分経っていた」という体験をする

### 利用シーン
- 通勤電車の中（片道30分）
- 昼休み（15分程度）
- 寝る前のリラックスタイム（20〜30分）
- 週末の暇な時間

## ペルソナ 2: 管理者/運営者

| 項目 | 内容 |
|------|------|
| **名前** | 鈴木 美咲（仮名） |
| **年齢** | 28歳 |
| **職業** | トレノケート マーケティング担当 |
| **ITリテラシー** | 中〜高（管理画面の操作に慣れている） |

### 動機
- コース情報を効率的にサービスに登録・更新したい
- ユーザーの利用状況を把握して、コンテンツ改善に活かしたい
- サービスの品質を維持し、正確なコース情報を提供したい

### 課題
- 1600以上のコースデータを手動で管理するのは大変
- コース情報の更新頻度が高い（新規追加、内容変更、終了）
- どのコースがユーザーに刺さっているか把握しにくい

### ゴール
- コースデータを効率的に一括登録・更新できる
- ユーザーの閲覧傾向を把握できる
- データの正確性を維持できる

### 利用シーン
- 新規コース追加時（月数回）
- コース情報更新時（随時）
- 利用状況レポート確認（週次）
