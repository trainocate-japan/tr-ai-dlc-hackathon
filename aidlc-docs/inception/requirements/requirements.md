## プロジェクト概要

### サービス名
**「ハマリーニング」** — IT/ビジネス研修コースのセレンディピティ発見プラットフォーム

> ハマる + ラーニング。没入して夢中になっているうちに、気がつけば学びが身についている体験。

### コンセプト
トレノケートの1600以上のIT/ビジネス研修コースを、TikTok風の縦無限スクロールUIで「時間を溶かしながら」ブラウズし、検索やレコメンドでは得られない偶然の出会い（セレンディピティ）を通じて、気がつけば学んでいるような体験を提供する。

### ビジョン
- 検索ではなく「眺める」「出会う」体験
- タイムパフォーマンスを無視した回り道の楽しさ
- 中毒性のあるUI/UXで夢中にさせる
- 結果的に学びにつながるセレンディピティ

## 機能要件（MVP）

### FR-1: コース無限スクロールフィード
- TikTok風の縦無限スクロールUIでコース情報を表示
- 各コースカードはビジュアルリッチに表示（カテゴリカラー、アイコン、キャッチコピー、月間ランキング順位）
- スワイプ/スクロールで次々とコースが表示される
- 表示順はランダム性を含む（完全なランダムではなく、カテゴリの多様性を保証）
- 「いいね」「あとで見る」アクションをカード上で実行可能
- コース情報カードとクイズカードの2種類が混在して表示される

### FR-2: コレクション＆発見ログ
- 「いいね」「あとで見る」したコースをコレクションとして保存
- 「あとで見る」は見終わったら解除可能
- 閲覧履歴を自動記録（3秒以上表示で記録）
- コレクションをカテゴリ別に振り返り可能
- 「あなたの発見マップ」— 閲覧・いいね済みコースをグラフ状にビジュアライズ
  - コースがノードとして表示され、同カテゴリのコース同士が線で繋がる
  - いいね済みコースは強調表示（色・サイズ）
  - 各ノードにコースカードの概要サマリーが表示され、過去の発見を思い出せる
  - 興味の広がりと繋がりを直感的に把握できる
- 学びの軌跡として振り返れるダッシュボード

### FR-3: ユーザー認証
- Cognito によるユーザー登録・ログイン
- ソーシャルログイン（Google）対応
- ログイン状態で履歴・コレクションを永続化

### FR-4: コースデータ管理
- トレノケートのコース情報をデータソースとして取得・格納
- コースデータファイルにはコースの内容・目的など全文が含まれる
- コース情報: コースコード、タイトル、内容全文、目的、カテゴリ、レベル、期間、対象者、関連スキル
- 管理者はS3バケットにコースデータファイル（CSV/JSON）を配置するだけで登録・更新
- コースコードが既存であれば更新、新規であれば追加登録
- 月間ランキングデータもS3バケット経由でバッチ取り込み
- 将来的にユーザー投稿（レビュー、おすすめコメント）を追加可能な設計

### FR-5: クイズ＆ポイントシステム
- コースごとにAI生成○×クイズカードがフィード内に表示される
- クイズはAmazon Bedrock（生成AIモデル）がコース情報を元に○×（マルバツ）2択形式で自動作成
- ユーザーはクイズに回答でき、正解するとポイントが加算される
- 累計ポイントに応じてユーザーレベルが上がる
- クイズ生成はイベント駆動で実行（コースデータ登録/更新時にDynamoDB Streams経由で自動トリガー、Token Limit Exception回避のため非同期）

### FR-6: Bedrockによるコンテンツ生成
- コースデータファイルにはコースの内容・目的など全文が含まれる
- カード表示用のコース概要（キャッチーなサマリー）をBedrockが全文からHTML形式で自動生成
- 生成されたHTMLは例（テンプレート）に習った形式で作成される
- 生成結果はDynamoDBのcourse-summariesテーブルにHTML形式で保存
- クイズ（問題文 + 正解○or× + 解説）をBedrockがコース全文から自動生成
- いずれもイベント駆動で実行（コースデータ登録/更新時にDynamoDB Streams → EventBridge → SQS → Lambda + Bedrockで非同期生成）

## 非機能要件（MVP）

### NFR-1: パフォーマンス
- フィードの初回表示: 2秒以内
- スクロール時の次コース読み込み: 200ms以内（プリフェッチ）
- 同時接続ユーザー: 100人（MVP段階）

### NFR-2: セキュリティ
- Cognito による認証・認可
  - パスワードポリシー: 16桁以上、大文字・小文字・数字必須
  - MFA: 任意（ユーザー選択）
  - その他詳細はConstruction Phaseで定義予定
- HTTPS通信必須
- API Gateway でのリクエスト認証
- DynamoDB のデータアクセス制御
- OWASP Top 10 対策

### NFR-3: スケーラビリティ
- サーバーレスアーキテクチャ（Lambda）による自動スケーリング
- DynamoDB のオンデマンドキャパシティ
- CloudFront によるフロントエンド配信

### NFR-4: ユーザビリティ
- モバイルファースト（スマートフォンでの縦スクロール体験を最優先）
- レスポンシブデザイン（PC/タブレットでも利用可能）
- 直感的な操作（チュートリアル不要）

### NFR-5: 可用性
- 99.9% 稼働率（AWSマネージドサービス活用）
- グレースフルデグラデーション（一部機能障害時も閲覧は継続可能）

## MVPスコープ定義

### MVP に含める（1〜2週間開発目標）
1. **コース無限スクロールフィード**（コア体験 — 情報カード＋クイズカード）
2. **コレクション＆発見ログ**（学び要素）
3. **クイズ＆ポイントシステム**（ゲーミフィケーション、Bedrock連携）
4. **ユーザー認証**（データ永続化に必須）
5. **コースデータ管理**（S3バケット経由バッチ登録・更新）
6. **月間ランキング取り込み**（S3バケット経由バッチ）

### MVP に含めない（将来拡張）
- ユーザー投稿（レビュー、コメント）
- ソーシャル機能（共有、フォロー）
- 管理画面UI（MVP段階はS3バケット配置のみ）
- プッシュ通知

---

## 制約事項

- コースデータの取得方法は社内システム連携で確定（S3バケット経由でデータファイル連携）
- コース情報の二次利用は許可済み
