# ユーザーストーリー

## 構成
- **分解アプローチ**: ハイブリッド（ユーザージャーニーベース + ペルソナ差分）
- **ジャーニーフロー**: 初回訪問 → ブラウズ（没入） → 発見・アクション → コレクション → 振り返り
- **アクセプタンス基準**: Given-When-Then形式（1〜2条件）
- **スコープ**: MVPのみ

## 成功KPI（定量的）
| 指標 | 目標値 |
|------|--------|
| 平均セッション時間 | 10分以上 |
| 1セッションあたりスクロール数 | 30コース以上閲覧 |
| 週次リピート率 | 40%以上 |
| コレクション登録率 | セッションの50%以上で1件以上登録 |

## ジャーニー 1: 初回訪問・認証

### US-001: ユーザー登録
**As a** 一般ユーザー  
**I want to** メールアドレスまたはGoogleアカウントで簡単に登録したい  
**So that** すぐにサービスを使い始められる

**アクセプタンス基準:**
- Given 未登録ユーザーがトップページにアクセスした When 「Googleで登録」ボタンを押す Then Googleアカウント認証後にフィード画面に遷移する
- Given 未登録ユーザーがメールで登録する When メールアドレスとパスワードを入力して送信する Then 確認メール送信後、認証完了でフィード画面に遷移する

**ペルソナ**: 一般ユーザー

---

### US-002: ログイン
**As a** 登録済みユーザー  
**I want to** メールアドレス/パスワードまたはGoogleアカウントで素早くログインしたい  
**So that** 前回の続きからサービスを利用できる

**アクセプタンス基準:**
- Given 登録済みユーザーがログイン画面にアクセスした When メールアドレス/パスワードまたは「Googleでログイン」で認証する Then フィード画面に遷移し、コレクションや履歴が復元される

**ペルソナ**: 一般ユーザー

---

## ジャーニー 2: ブラウズ（没入体験）

### US-003: コースフィードの無限スクロール
**As a** 一般ユーザー  
**I want to** TikTokのように縦にスクロールして次々とコース情報を眺めたい  
**So that** 目的なく没入的にブラウズして偶然の出会いを楽しめる

**アクセプタンス基準:**
- Given ログイン済みユーザーがフィード画面にいる When 下にスクロールする Then 新しいコースカードが途切れなく表示される
- Given フィード画面を表示した When 2秒以内 Then 最初のコースカードが表示される

**ペルソナ**: 一般ユーザー

---

### US-004: コースカードのビジュアル表示
**As a** 一般ユーザー  
**I want to** 各コースがビジュアルリッチなカードで表示され、人気度もわかるようにしてほしい  
**So that** 一目で興味を引かれるかどうか判断でき、眺めるだけで楽しい

**アクセプタンス基準:**
- Given コースカードが表示された When ユーザーが見る Then カテゴリカラー、コースタイトル、概要、レベル、対象者、月間ランキング順位が視覚的に表示される

**ペルソナ**: 一般ユーザー

---

### US-005: フィードの多様性保証
**As a** 一般ユーザー  
**I want to** 同じカテゴリのコースばかりが連続しないでほしい  
**So that** 予想外のジャンルに出会えて飽きない

**アクセプタンス基準:**
- Given フィードをスクロールしている When 連続する5枚のカードを見る Then 少なくとも2つ以上の異なるカテゴリが含まれる

**ペルソナ**: 一般ユーザー

---

## ジャーニー 3: 発見・アクション

### US-006: コースに「いいね」する
**As a** 一般ユーザー  
**I want to** 気になったコースに「いいね」をつけたい  
**So that** 後で見返せるようにマークできる

**アクセプタンス基準:**
- Given コースカードが表示されている When 「いいね」ボタンをタップする Then いいねが記録され、ボタンの見た目が変わる

**ペルソナ**: 一般ユーザー

---

### US-007: コースを「あとで見る」に追加する
**As a** 一般ユーザー  
**I want to** 興味はあるが今は詳しく見ないコースを「あとで見る」に保存したい  
**So that** フィードの没入体験を中断せずに気になるコースを記録できる

**アクセプタンス基準:**
- Given コースカードが表示されている When 「あとで見る」ボタンをタップする Then コースがあとで見るリストに追加され、フィードの表示は中断されない

**ペルソナ**: 一般ユーザー

---

### US-008: コース詳細の表示
**As a** 一般ユーザー  
**I want to** 気になったコースの詳細情報を見たい  
**So that** コースの内容をもう少し深く知ることができる

**アクセプタンス基準:**
- Given コースカードが表示されている When カードをタップする Then コースの詳細情報（期間、前提知識、カリキュラム概要等）が表示される

**ペルソナ**: 一般ユーザー

---

## ジャーニー 4: コレクション管理

### US-009: コレクション一覧の表示
**As a** 一般ユーザー  
**I want to** 「いいね」や「あとで見る」したコースを一覧で見たい  
**So that** 自分が興味を持ったコースを振り返れる

**アクセプタンス基準:**
- Given コレクション画面にアクセスした When 表示される Then 「いいね」と「あとで見る」のコースがそれぞれリスト表示される
- Given 「あとで見る」リストのコースを確認した When 「見終わった」操作をする Then そのコースが「あとで見る」リストから外れる

**ペルソナ**: 一般ユーザー

---

### US-010: コレクションのカテゴリ別表示
**As a** 一般ユーザー  
**I want to** コレクションをカテゴリ別にフィルタして見たい  
**So that** 特定分野への興味の深さを把握できる

**アクセプタンス基準:**
- Given コレクション画面にいる When カテゴリフィルタを選択する Then 選択したカテゴリのコースのみが表示される

**ペルソナ**: 一般ユーザー

---

## ジャーニー 5: 振り返り（発見ログ）

### US-011: 閲覧履歴の自動記録
**As a** 一般ユーザー  
**I want to** 閲覧したコースが自動的に記録されてほしい  
**So that** 意識せずに自分の興味の軌跡が残る

**アクセプタンス基準:**
- Given フィードでコースカードを閲覧した When 一定時間（3秒以上）表示された Then そのコースが閲覧履歴に自動記録される

**ペルソナ**: 一般ユーザー

---

### US-012: 発見マップの表示
**As a** 一般ユーザー  
**I want to** 自分が閲覧したカテゴリの広がりを可視化して見たい  
**So that** 「こんなに幅広く見ていたんだ」と気づき、学びの実感を得られる

**アクセプタンス基準:**
- Given 発見マップ画面にアクセスした When 表示される Then 閲覧済みカテゴリが視覚的に（チャートやマップで）表示され、カテゴリごとの閲覧数がわかる

**ペルソナ**: 一般ユーザー

---

## ジャーニー 6: コースデータ管理（管理者）

### US-013: コースデータの一括登録
**As a** 管理者  
**I want to** S3バケットにコースデータファイル（CSV/JSON）を置くだけでコースを一括登録したい  
**So that** 1600以上のコースを最小限の手間でシステムに投入できる

**アクセプタンス基準:**
- Given 管理者がコースデータファイルをS3バケットに配置した When バッチ処理が実行される Then 全コースがシステムに登録され、登録結果レポートが出力される

**ペルソナ**: 管理者/運営者

---

### US-014: コースデータの更新
**As a** 管理者  
**I want to** 更新したコースデータファイルをS3バケットに置くだけでコース情報を更新したい  
**So that** コース内容の変更を迅速にサービスに反映できる

**アクセプタンス基準:**
- Given 管理者が更新データファイルをS3バケットに配置した When バッチ処理が実行される Then コースコードが既存のものは更新され、新規のものは追加登録される

**ペルソナ**: 管理者/運営者

---

## ジャーニー 7: クイズ＆ポイント

### US-015: クイズカードの表示と回答
**As a** 一般ユーザー  
**I want to** フィード内にコース関連のクイズカードが表示され、その場で答えたい  
**So that** 楽しみながら知識が定着し、没入体験がさらに深まる

**アクセプタンス基準:**
- Given フィードをスクロールしている When クイズカードが表示される Then コースに関連するクイズが表示され、選択肢をタップして回答できる
- Given クイズに回答した When 正解/不正解が判定される Then 結果がその場で表示される

**ペルソナ**: 一般ユーザー

---

### US-016: ポイント加算とユーザーレベル
**As a** 一般ユーザー  
**I want to** クイズに答えるとポイントが貯まり、ユーザーレベルが上がってほしい  
**So that** 続けるモチベーションが生まれ、成長を実感できる

**アクセプタンス基準:**
- Given クイズに正解した When ポイントが加算される Then 累計ポイントとユーザーレベルが更新され、画面に反映される

**ペルソナ**: 一般ユーザー

---

### US-017: クイズの自動生成（Bedrock）
**As a** 管理者  
**I want to** コース情報を元にクイズがAI（Amazon Bedrock）で自動生成されてほしい  
**So that** 手動でクイズを作成する手間なく、全コースにクイズを提供できる

**アクセプタンス基準:**
- Given コースデータが登録されている When クイズ生成バッチが実行される Then 各コースに対して生成AIモデルがクイズ（問題文 + 選択肢 + 正解）を自動作成し、保存される

**ペルソナ**: 管理者/運営者

---

### US-018: 月間ランキングデータの取り込み
**As a** 管理者  
**I want to** 別途収集している月間人気ランキングデータをバッチでシステムに反映したい  
**So that** ユーザーがコースカードで人気度を確認できる

**アクセプタンス基準:**
- Given 管理者がランキングデータファイルをS3バケットに配置した When バッチ処理が実行される Then 各コースに月間ランキング順位が紐づけられ、フィードのカード表示に反映される

**ペルソナ**: 管理者/運営者

## ペルソナ差分

### 一般ユーザー固有の体験
- フィードの没入体験が最重要（US-003〜US-005）
- アクションは「片手で、フィードを止めずに」実行できること（US-006, US-007）
- 振り返りは「驚き」を伴う体験であること（US-012）

### 管理者固有の体験
- 効率性が最重要（一括操作、バッチ処理）
- MVP段階ではCLI/スクリプトベースで十分（管理画面UIは将来）
- データの正確性と整合性の担保
