### Question 1: ユニット分割の粒度
システムをどの粒度でユニットに分割しますか？

A) 3ユニット（フロントエンド / バックエンドAPI / バッチ処理＆インフラ）
B) 4ユニット（フロントエンド / バックエンドAPI / バッチ処理 / インフラ(IaC)）
C) 5ユニット以上（サービスごとに独立: Feed / Collection / Quiz / Discovery / Batch / Infra / Frontend）
D) おまかせ（最適な分割を提案してほしい）
X) その他（[Answer]: タグの後に詳細を記述してください）

### Question 2: IaC（インフラ定義）の扱い
インフラのコード化はどのように管理しますか？

A) 独立ユニット（インフラだけで1つのユニット、CDKスタックとして管理）
B) 各ユニットに含める（フロントエンドユニットにCloudFront/S3、APIユニットにAPI GW/Lambda等）
C) 全体を1つのCDKプロジェクトで管理し、スタック分割で対応
X) その他（[Answer]: タグの後に詳細を記述してください）
